# 适读人群

本书主要写给三类读者：正在系统学习 Android 的初学者、准备从旧式 Android 开发迁移到现代实践的开发者，以及需要一套结构化课程材料的教师或团队导师。不同读者的起点不同，但他们面临的共同问题是一样的：信息来源分散、术语很多、旧教程和新实践混杂，难以形成稳定的知识结构。

## 适合哪些读者

- 已具备基础编程能力，希望从零建立 Android 全局认知的读者。
- 会一些 Java 或旧式 Android API，但希望转向 Kotlin、AndroidX 和 Jetpack 的读者。
- 想把零散经验整理成系统知识，用于教学、培训或团队入门的读者。

## 不要求的前置条件

- 不要求已经掌握 Kotlin。
- 不要求已经做过 Android 项目。
- 不要求已经理解架构、网络、数据库或并发的完整体系。

## 更适合你的阅读方式

如果你是完全初学者，建议严格按目录顺序阅读，先建立整体框架，再进入每一部分的实践。如果你来自旧项目维护场景，则可以重点阅读基础篇、数据存储、架构、并发与系统组件部分，把“历史写法”和“当前推荐方案”的边界先梳理清楚。

如果你是教师、培训讲师或团队导师，这本书更适合被当成“课程母本”而不是“逐页照念讲稿”。也就是说，你可以按部分拆成阶段课程，但仍建议保留它原本的前后依赖关系，而不要把生命周期、状态管理、后台任务和发布流程拆得彼此完全独立。

## 你将从本书得到什么

本书不会把 Android 开发讲成一堆互不相干的 API，也不会只停留在“把界面做出来”的层面。它更关注三件事：

- 让你知道一个 Android 应用是如何被系统组织起来的。
- 让你知道今天的新项目应该默认采用什么做法。
- 让你具备从章节知识过渡到真实工程结构的能力。

但它不会替代真实项目实践。更准确地说，本书能帮助你建立更稳定的判断框架，而真正让这些框架变成能力的，仍然是你在练习项目、课程作业或团队项目中的反复验证。

## 小结

如果你希望得到的是一套从平台认知、界面开发、数据处理到工程实践逐步展开的学习路径，这本书适合你。如果你期待的是某一个零散 API 的速查手册，它并不是为那个目标写的。
