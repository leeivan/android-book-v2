# 前言

Android 学习资料从来不缺，真正缺的是一条足够稳定的主线。很多读者第一次接触 Android 时，会在旧教程、新 API、零散博客、过时视频和项目片段之间来回切换，结果学到的是大量彼此不连贯的知识点，却很难形成“一个现代 Android 应用到底如何被组织起来”的整体认知。也正因为如此，本书并不想再做一本以 API 罗列为中心的教程，而是希望重建一条更接近今天 Android 实践的学习路径。

这本书的起点很明确：以 Kotlin、AndroidX、Jetpack 和现代工程实践为正文主线，把 Activity、Fragment、ViewModel、Room、Flow、WorkManager、通知、权限、构建和发布等内容放回同一套工程语境里理解。换句话说，本书更关心“能力如何协作”，而不仅仅是“API 长什么样”。如果读者只能记住某个类的用法，而不能判断它为什么应该放在这一层、为什么今天推荐这样设计，那么学习仍然是不完整的。

在写作组织上，本书尽量坚持三条原则。第一，章节之间是连续的，而不是彼此独立的词条。第二，正文默认采用今天新项目更应优先选择的方案，历史写法只在理解遗留项目和迁移语境时出现。第三，每一章都尽量把概念、实践任务、常见误区和工程判断放到同一页面中，让读者不仅“看懂”，还知道“该怎样做、为什么这样做、容易在哪出错”。

这意味着你在阅读本书时，得到的不应该只是零散结论，而是一套逐步展开的结构：平台和工程基础如何建立，UI 和交互如何组织，数据从哪里来又怎样稳定进入页面，架构与并发如何承接长期复杂度，系统组件和发布流程又怎样把应用真正带到用户设备上。到了案例部分，前面所有主线会重新被串起来，帮助你把“知道知识点”推进到“能组织真实项目”。

这本书同样试图照顾两类特别常见的读者。一类是刚进入 Android 的学习者，他们需要的是一条不被旧资料和新资料反复拉扯的主线；另一类是已经接触过 Java 或旧式 Android API 的开发者，他们真正需要的往往不是从头再学一遍，而是重新梳理“哪些仍然重要、哪些应当迁移、今天默认方案是什么”。因此，本书会保留一定的历史视角，但不会再把历史写法当成新项目主线。

如果你愿意按目录顺序读下去，并且愿意在每个阶段都配合最小实践任务，这本书最希望带给你的，不只是“会做几个页面”或“会调几个接口”，而是逐渐建立一种更稳定的判断力：当你面对一个 Android 需求时，知道它应该放到哪一层，依赖什么主线能力，怎样避开明显过时或脆弱的设计，以及如何把它组织成一个可以继续增长的工程。

## 本书定位

- 面向现代 Android 主线，而不是历史 API 汇编。
- 面向系统学习和工程落地，而不是零散速查。
- 兼顾初学者、旧项目迁移者和课程教学场景。

## 你会在这本书中反复看到的关键词

- 页面状态比页面实例更重要。
- 数据应通过清晰边界进入 UI。
- 系统组件是平台边界，不是普通对象替代品。
- 工程实践和发布流程从来不是“后面再说”的附加项。

## 小结

如果本书能帮助你建立一种更接近今天 Android 实践的整体视角，让你从“看过很多知识点”走到“能把知识点组织成工程”，那么它就达到了写作目的。
