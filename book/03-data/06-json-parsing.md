# JSON 解析

JSON 本身不属于 Android 专有知识，但它几乎无处不在：网络返回、离线缓存、配置文件、调试输出都可能出现它。学习 JSON 解析的重点，不是死记某个库的 API，而是理解结构化数据如何从字符串变成应用可用对象。

## 学习目标

- 理解 JSON 的结构特点。
- 理解 JSON 与网络层、本地缓存之间的联系。
- 为后续网络开发章节提供数据格式基础。

## 前置知识

- 已具备数据存储分类的基础认知。
- 知道结构化数据需要映射到应用模型中。

## 正文

### 1. 为什么 JSON 几乎绕不开

Android 应用今天和远程服务交互非常频繁，而 JSON 是最常见的数据交换格式之一。它常见的原因在于：

- 结构直观。
- 跨语言可读性强。
- 适合表达列表、对象和嵌套结构。

因此，学习 Android 网络开发时，如果没有 JSON 基础，就很难真正理解接口数据是如何流入界面的。

### 2. JSON 解析真正要解决什么问题

不是“把文本转成对象”这么简单，而是：

- 字段名和模型字段如何对应。
- 可选字段和空值如何处理。
- 列表和嵌套结构如何表达。
- 错误数据和缺失字段如何防御。

这些问题在网络层和本地缓存中都会反复出现。

### 3. 为什么不应把解析逻辑散落在界面层

如果界面直接处理 JSON 文本，后面会很快出现几个问题：

- 可读性差。
- 模型不稳定。
- 错误处理难以集中。

更合理的做法是让 JSON 解析发生在网络层或数据层，再把已经结构化的结果交给界面层。

### 4. JSON 与本地数据层的关系

JSON 经常是远程接口的输入输出格式，但最终应用内部仍然需要更稳定的数据模型。也就是说：

- JSON 更像交换格式。
- 应用模型更像内部结构。

理解这一点，能帮助你避免把“接口字段”直接无脑传播到整个应用架构中。

### 5. 最小实践任务

1. 写出一个最简单的新闻列表 JSON 样例。
2. 标出其中的对象、数组和可选字段。
3. 思考：如果某个字段缺失，界面应如何处理。

### 6. 常见误区

- 把 JSON 当成“只是字符串”。
- 直接在界面里做解析。
- 把接口字段结构直接等同于应用内部模型。

## 小结

JSON 解析是网络开发和数据层之间的重要桥梁。学会它的关键不在于某个库的写法，而在于理解数据如何从交换格式变成应用可维护的内部结构。
