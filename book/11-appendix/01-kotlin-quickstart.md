# Kotlin 快速入门

这本书把 Kotlin 作为 Android 开发的默认语言，但这并不意味着你需要先成为 Kotlin 专家才能继续往下学。本附录的目的，是帮你以最短路径建立足够用的 Kotlin 基础，让你能读懂示例、写出简单代码，并在后续章节中逐步加深理解。它不是完整语言教材，而是一份“为了学 Android，哪些 Kotlin 能力必须尽快站稳”的速查入口。

## 学习目标

- 快速理解 Kotlin 中最常见的语法和类型系统特征。
- 知道哪些 Kotlin 能力会在 Android 开发中高频出现。
- 建立“先会读写，再深入语言特性”的学习顺序。
- 为阅读 ViewModel、Flow、Room、Compose 等章节代码打基础。

## 前置知识

- 具备任意一门面向对象语言基础。
- 知道变量、函数、类和集合这些基本概念。

## 正文

### 1. 为什么 Android 学习要优先 Kotlin

对当前 Android 开发而言，Kotlin 已经不只是“可选语言”，它更像默认入口。很多现代示例、文档和 Jetpack 能力都优先围绕 Kotlin 提供最佳体验。

这意味着你即使来自 Java 背景，也应尽快建立 Kotlin 的阅读和表达能力。否则，后面很多现代 Android 代码会在你眼里只剩“语法看不懂”，而不是“架构没理解”。

### 2. 最值得先掌握哪些能力

对 Android 初学者来说，最优先掌握的是：

- `val` 和 `var` 的区别。
- 可空类型与空安全调用。
- 函数定义和默认参数。
- 数据类。
- 集合处理的基本写法。
- lambda 和高阶函数的基本阅读能力。

这些能力会在 ViewModel、Room、网络返回模型、列表处理和状态管理中反复出现。

### 3. 为什么空安全这么重要

Kotlin 最大的现实价值之一，是把很多过去运行时才暴露的空引用问题前移到编译期思考。对 Android 而言，这特别重要，因为：

- 页面参数可能为空。
- 系统回调不总是稳定。
- 网络字段可能缺失。
- 异步状态经常处于“还没到”的中间态。

如果你没有建立可空类型意识，后面几乎所有章节都会读得很吃力。

### 4. 不可变性为什么在 Android 中很有价值

Kotlin 鼓励优先使用不可变值，这一点对 Android 特别重要，因为界面状态和异步数据非常容易在多人、跨层或跨线程流动时变乱。

更偏向不可变的写法，通常会让：

- 状态更容易追踪。
- 页面更新更清晰。
- 并发问题更少。

这也是为什么现代 Android 代码里会大量出现 `val`、不可变数据类和状态复制写法。

### 5. 如何学习 Kotlin 才不容易卡住

最有效的方法通常不是先完整学一遍语言手册，而是：

1. 先掌握最常见的语法。
2. 在 Android 例子里反复看这些语法怎么用。
3. 遇到不理解的语言特性再回头补。

这样学习更接近实际开发路径，也更容易保持节奏。

### 6. 哪些高级特性可以暂时放后面

像委托、内联、复杂泛型、DSL 设计、上下文接收器等内容都重要，但不需要在入门阶段一次吃透。只要你先把：

- 空安全。
- 类与函数。
- 数据类。
- 集合与 lambda。
- 基础协程语法。

站稳，已经足够读完整本书的大部分内容。

### 7. 一个更实用的学习目标：先能读，再能写，再谈优雅

很多人学 Kotlin 时过早追求“写得像高手”，结果反而会卡住。更稳妥的目标顺序是：

1. 先读懂常见 Kotlin 代码。
2. 再写出简单而正确的 Kotlin 代码。
3. 最后再去优化表达、抽象和风格。

这会让你更快进入 Android 正文学习，而不是长期停在语言门口。

### 8. 最小实践任务

1. 用 Kotlin 写一个简单的数据类表示待办事项。
2. 写一个函数，根据完成状态返回不同显示文本。
3. 练习用安全调用和空判断处理一个可能为空的标题字段。
4. 把一段你熟悉的 Java 小逻辑改写成 Kotlin，并比较样板代码差异。

### 9. 常见误区

- 认为必须先完整掌握 Kotlin 才能学 Android。
- 只背语法，不在真实场景里使用。
- 忽略可空类型，沿用旧的“先写再说”思路。
- 一开始就追求复杂语法和“高级写法”。

## 小结

Kotlin 入门阶段最重要的不是广度，而是建立稳定的阅读和编码能力。只要你先掌握最常用的少数能力，后面整本书都会更顺畅。
