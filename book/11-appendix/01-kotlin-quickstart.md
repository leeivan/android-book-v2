# Kotlin 快速入门

本书虽然不是 Kotlin 语法教材，但现代 Android 开发已经很难绕开 Kotlin。附录里的这份“快速入门”不是要取代系统学习，而是帮助已经具备编程基础、正在学习 Android 的读者尽快抓住最常用、最有价值的一部分语法和思维方式，避免在阅读后续代码时被细节绊住。

真正值得优先掌握的，不是全部语言特性，而是那些会反复出现在 Android 项目里的核心能力：空安全、数据类、扩展函数、`when`、高阶函数、协程相关语法，以及可读性更强的不可变风格。

## 学习目标

- 快速建立阅读 Kotlin Android 代码所需的最小语法基础。
- 理解 Kotlin 相比 Java 在 Android 中最常见的优势点。
- 抓住空安全、数据类、扩展函数和函数式风格这几条主线。
- 为阅读 ViewModel、Flow、Room 和现代 API 打基础。

## 前置知识

- 已有至少一门主流语言的基础编程经验。
- 愿意在阅读 Android 代码时接受 Kotlin-first 作为主线。

## 正文

### 1. 先把 Kotlin 当成“更适合现代 Android 的默认语言”

Android 官方已经长期明确推荐 Kotlin 作为首选语言。对学习者来说，这意味着最重要的不是纠结“要不要完全抛弃 Java”，而是先接受一个现实：现代 Android 文档、示例和生态，绝大多数都会以 Kotlin 为主。

因此，这份附录的目标不是比较语言优劣，而是帮助你尽快进入现代 Android 主线。

### 2. 最先该掌握的是空安全

Kotlin 最容易让 Java 开发者立刻感受到差异的，就是空安全。它通过类型系统明确区分：

- 默认不可空类型。
- 显式可空类型。

这会直接改变你写 Android 代码的方式，因为页面状态、网络结果、数据库查询和可选参数都经常涉及空值处理。只要空安全思维建立起来，很多空指针风险会在编译期就暴露出来。

### 3. 数据类让模型表达更轻量

Android 项目里会大量出现 DTO、Entity、UI model、状态对象。Kotlin 的 `data class` 能大幅减少样板代码，让这些模型更专注于表达数据本身，而不是重复写 `equals`、`hashCode`、`toString` 和构造器。

这也是为什么从网络层到 ViewModel 状态层，Kotlin 模型代码通常会比传统 Java 简洁很多。

### 4. 不可变默认值和表达式风格很适合状态建模

Kotlin 更鼓励：

- 使用 `val` 而不是默认到处可变。
- 用表达式而不是冗长分支。
- 用 `when`、`let`、`run` 等更清晰组织逻辑。

这种风格特别适合 Android 页面状态建模，因为状态本身就更适合被“产生”和“替换”，而不是被很多地方随意修改。

### 5. 扩展函数和高阶函数为什么在 Android 代码里到处都是

Kotlin 让你可以为已有类型增加扩展函数，这对 Android 非常有价值。因为你经常会遇到一些“想给某类对象增加通用辅助能力，但又不想写工具大类”的场景。

高阶函数同样重要，因为 Flow、集合操作、Compose 和协程 API 都大量依赖函数式风格。如果只把 Kotlin 当成“Java 换皮”，会很难真正读懂现代 Android 代码。

### 6. 协程相关语法是阅读现代 Android 代码的门槛之一

即使这一附录不深入讲并发，也必须提醒一点：`suspend`、lambda、`when`、扩展函数和不可变状态对象，是阅读协程和 Flow 代码的基本前提。你不必一开始把所有语法吃透，但至少要看懂这些代码结构在表达什么。

### 7. 实践任务

起点条件：

- 已准备阅读或编写 Kotlin Android 代码。

步骤：

1. 用 Kotlin 写一个最小数据类和状态类。
2. 练习把几个可空字段改写成更明确的空安全处理。
3. 用 `when` 改写一段原本依赖多层 `if` 的分支。
4. 尝试阅读一个包含 `StateFlow` 和 `data class` 的 ViewModel 代码片段。

预期结果：

- 你能更自然地读懂本书后续大多数 Kotlin 示例。
- 你会把 Kotlin 理解成状态驱动 Android 的自然语言，而不是附加语法层。

自检方式：

- 你能解释：Kotlin 空安全对 Android 有什么现实价值。
- 你能判断：什么时候用 `data class` 特别合适。
- 你能说出：为什么不可变思维有助于状态管理。

调试提示：

- 如果你总想把 Kotlin 完全按 Java 思维翻译，阅读现代代码会非常吃力。
- 如果你对空安全不敏感，后面网络和状态代码会频繁出问题。

## 小结

Kotlin 快速入门的重点，不是覆盖全部语法，而是让你抓住那些真正反复出现在现代 Android 项目里的能力。只要空安全、数据类和状态友好的表达方式建立起来，后面的大部分代码都会顺畅很多。
