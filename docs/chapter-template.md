# 章节模板

```md
# 章节标题

开篇 2 到 4 段：说明本章要解决的真实问题、为什么重要，以及它与前后章节的关系。

## 学习目标

- 目标 1
- 目标 2
- 目标 3

## 前置知识

- 前置知识 1
- 前置知识 2

## 正文

### 1. 问题背景与学习动机

用自然段解释本章出现的原因、读者会遇到的真实问题，以及本章的目标。

### 2. 核心概念与判断框架

解释概念、边界、适用场景和常见误解。不要只列定义或 API 名称。

### 3. 推荐做法与最小示例

先说明为什么采用这种做法，再给出最小可运行示例。

```kotlin
// 示例代码
```

说明：

- 代码放在哪里。
- 关键语句为什么这样写。
- 运行后应看到什么。

### 4. 实践任务

1. 步骤 1
2. 步骤 2
3. 步骤 3

预期结果：

- 结果 1
- 结果 2

自检方式：

- 自检点 1
- 自检点 2

调试提示：

- 提示 1
- 提示 2

### 5. 常见误区

- 误区 1
- 误区 2

## 小结

用 1 到 2 段总结本章收获，并说明下一章如何承接。

## 练习题

1. 概念理解题
2. 编码实现题
3. 扩展思考题

## 参考资料

- 官方文档 1
- 官方文档 2
```

除学习目标、前置知识、常见误区和练习题外，不要让正文长期停留在条目化状态。
