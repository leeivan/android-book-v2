# 第9章 本地存储、文件访问与媒体能力

移动应用的数据并不只来自网络，也不只存在内存中。本章聚焦本地存储与文件访问，帮助读者建立“什么数据该放哪里、如何合规访问共享内容”的整体判断。

## 学习目标

- 理解键值配置、文件数据和共享媒体访问的区别。
- 理解 DataStore、应用专属目录、MediaStore、SAF 与 Photo Picker 的适用场景。
- 理解现代 Android 存储模型与权限边界。
- 能够设计一条合规且清晰的本地文件访问路径。

## 9.1 键值型配置

### 9.1.1 DataStore 作为主线

- 新项目优先使用 DataStore 处理轻量配置。
- 需要明确区分 Jetpack DataStore 与 Preference UI 中的 `PreferenceDataStore` 接口。

### 9.1.2 SharedPreferences 的位置

- 仍会在旧项目中见到。
- 适合理解历史实现，但不再作为正文默认方案。

## 9.2 文件与应用专属目录

### 9.2.1 私有文件

- 放在应用专属目录。
- 适合缓存、导出草稿、业务附件等私有数据。

### 9.2.2 为什么不要默认申请广泛存储权限

- 平台已经转向更严格的最小权限模型。
- 设计文件访问方案时应优先考虑最小暴露和最小授权。

## 9.3 共享媒体访问

### 9.3.1 MediaStore

- 适用于访问共享图片、视频和音频。
- 更符合现代 Android 存储规则。

### 9.3.2 Photo Picker

- 当需求只是“让用户选图或选视频”时，应优先考虑 Photo Picker。
- 它通常比申请广泛媒体权限更合适。

### 9.3.3 SAF

- 适合通用文件和目录选择。
- 是处理文档型访问的重要工具。

## 9.4 本章实践任务

- 使用 DataStore 保存一个应用设置项。
- 通过系统选择器让用户选择一个文件。
- 通过 Photo Picker 选择一张图片并展示预览。

## 9.5 小结

本章的核心不是 API 数量，而是判断能力：知道什么时候用配置存储，什么时候用私有文件，什么时候通过平台受控接口访问共享媒体。
