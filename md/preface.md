# 前言

## 为什么重写

旧版内容覆盖面很广，但存在一个明显问题：很多章节已经意识到 Android 平台在权限、后台执行、存储、通知和定位等方面发生了较大变化，却仍然把不少历史 API 放在正文主线中讲解。对于刚入门的读者来说，这会造成认知混乱：书中明明提示某些方案已经不再推荐，但真正一步一步带着读者写下去的，仍然是这些旧方案。

因此，v2 版的目标不是简单修补，而是调整整本书的叙事顺序与技术基线：

- 将现代 Android 工程实践放到正文主线。
- 把历史 API 收纳为迁移知识，而不是默认知识。
- 在章节之间建立统一的项目方法，而不是孤立罗列控件和接口。

## 面向读者

本书默认读者已经具备至少一种编程语言的基础语法能力，能够理解变量、函数、类、对象和集合等基本概念。对 Kotlin 没有深入经验并不会构成障碍，书中会优先使用清晰、稳定的 Kotlin 写法，并尽量减少为“技巧”而写的代码。

## 写作原则

- 先讲问题，再讲方案，最后讲实现。
- 先给推荐路径，再补历史路径和迁移说明。
- 先建立设计决策，再展示 API 用法。
- 每章给出最小实践任务，避免只形成概念记忆。

## 如何使用仓库

本仓库以 Markdown 书稿为主，适合在线阅读与持续迭代。章节文件放在 `md/` 目录下，README 负责提供入口链接。后续如果加入示例代码，建议采用“书稿 + samples 子目录”的方式，保持阅读内容与可运行项目在同一仓库中协同维护。
