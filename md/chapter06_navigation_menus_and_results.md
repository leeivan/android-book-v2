# 第6章 导航、菜单、应用栏与结果回调

一个应用是否易用，很大程度上取决于页面之间如何切换、顶部操作如何组织，以及结果如何回传。旧版 Android 教程常把这些内容拆散讲解，本章则把它们统一放在“页面流转与用户路径”这条主线上。

## 学习目标

- 理解应用内导航的基本概念。
- 理解应用栏、菜单与页面层级之间的关系。
- 理解 Activity Result API 作为现代结果回调机制的角色。
- 能够设计一个简单但结构清晰的页面流。

## 6.1 导航的基本问题

### 6.1.1 页面层级与返回路径

- 用户当前在哪里。
- 用户能做什么。
- 用户如何回到上一层或切换目标页面。

### 6.1.2 导航不只是“打开新页面”

- 导航同时涉及返回栈、状态保存和深层链接入口。

## 6.2 应用栏与菜单

### 6.2.1 MaterialToolbar 作为主线

- 现代项目中通常使用 Toolbar 或 MaterialToolbar。
- 应用栏承担标题、返回、主要操作和溢出操作入口。

### 6.2.2 菜单的职责边界

- 全局操作。
- 页面相关操作。
- 不应把所有按钮都塞进溢出菜单。

## 6.3 结果回调

### 6.3.1 为什么不再以 onActivityResult 作为主线

- 历史 API 仍会遇到，但不再适合作为新项目默认方案。
- Activity Result API 能更好地表达“发起动作”和“接收结果”的对应关系。

### 6.3.2 常见结果场景

- 选择图片。
- 选择联系人。
- 请求权限。
- 打开系统选择器。

## 6.4 本章实践任务

- 构建一个包含列表页和详情页的基本导航流。
- 为页面配置应用栏和菜单操作。
- 使用 Activity Result API 完成一次外部结果回传。

## 6.5 小结

导航、菜单和结果回调本质上都在回答一个问题：用户的操作如何在页面之间形成连续流程。把这条流程设计清楚，应用的可用性就会明显提升。
