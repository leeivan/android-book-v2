# 第1章 现代 Android 开发概览与环境搭建

本章是全书的入口。读者需要先建立两个判断：第一，Android 开发已经不是只会写几个 Activity 就足够的时代；第二，平台、商店规则、权限模型和构建系统会直接影响应用的设计方式。因此，本章既介绍平台概况，也明确本书采用的技术基线。

## 学习目标

- 理解 Android 平台、API Level 与 `targetSdk` 的关系。
- 完成 Android Studio、SDK、模拟器和真机调试环境搭建。
- 知道本书为何以 Kotlin 和 AndroidX 作为默认基线。
- 运行第一个应用，并观察日志、布局与项目结构。

## 1.1 现代 Android 开发的基本判断

### 1.1.1 平台、版本与 API Level

- Android 版本名只是外在标签，真正影响代码的是 API Level 和平台行为变更。
- `minSdk` 决定可支持的最低系统版本，`targetSdk` 决定应用是否按新规则运行。
- 学习 Android 时，重点不应放在记忆全部版本名，而应放在理解权限、后台限制、存储规则和组件导出等行为变化。

### 1.1.2 Google Play 与发布约束

- 新项目不仅要能运行，还要能通过商店审核、适配新系统、满足权限与隐私要求。
- 开发中的很多设计选择，其实最终会受到 `targetSdk`、商店政策和系统安全模型约束。

### 1.1.3 本书的统一技术基线

- 语言默认采用 Kotlin。
- 组件体系默认采用 AndroidX。
- 界面主线采用 View/XML，并在相关章节补充 Compose 视角。
- 状态管理默认采用 ViewModel。

## 1.2 开发环境搭建

### 1.2.1 Android Studio 与 SDK

- 安装 Android Studio 稳定版。
- 配置 SDK Platform、Build Tools、Platform Tools 和 Emulator。
- 了解 JDK、Gradle、AGP 在工程中的角色划分。

### 1.2.2 模拟器与真机调试

- 创建至少一个手机模拟器。
- 学会使用 Logcat、Device Manager、Running Devices。
- 配置真机 USB 调试，确认真机和模拟器都能运行项目。

### 1.2.3 常见环境问题

- 虚拟化未开启导致模拟器性能差。
- SDK 路径错误导致 Gradle 同步失败。
- 设备驱动、ADB 授权和网络代理导致真机不可用。

## 1.3 第一个应用

### 1.3.1 创建项目

- 选择 Empty Views Activity。
- 使用 Kotlin 作为默认语言。
- 观察 `app/src/main`、`res`、`AndroidManifest.xml` 和 Gradle 脚本。

### 1.3.2 运行与调试

- 在模拟器和真机上运行应用。
- 使用 Logcat 观察启动日志和异常信息。
- 初步理解编译、安装、启动这条最小工作链路。

### 1.3.3 从第一个应用看项目结构

- Activity 是入口点之一，但不是项目的全部。
- 资源、清单、依赖与构建脚本同样是 Android 项目的核心组成部分。

## 1.4 本章实践任务

- 搭建完整开发环境。
- 创建并运行一个空白 Kotlin 工程。
- 修改界面文案并重新运行，确认编辑到构建再到安装的闭环。

## 1.5 小结

本章的重点不是把环境配置步骤机械记住，而是建立“现代 Android 开发是平台规则 + 工程结构 + 运行环境共同作用”的整体认识。后续章节都建立在这一前提之上。
