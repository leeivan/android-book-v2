# 第8章 网络通信与远程数据获取

旧版教材常把 Android 开发讲成“本地界面与本地 API”的集合，但真实应用几乎一定会与远程数据打交道。本章新增网络主线，帮助读者理解现代移动应用最基本的数据获取方式。

## 学习目标

- 理解移动应用中的 HTTP、REST 和 JSON 基本概念。
- 理解 Retrofit 和 OkHttp 在常见网络层中的角色。
- 理解网络请求、错误处理和界面状态之间的关系。
- 能够完成一次最小的远程数据读取与展示。

## 8.1 网络层的基本问题

### 8.1.1 为什么需要单独一章网络

- 真实应用很少只依赖本地静态数据。
- 很多架构设计、缓存策略和错误处理都围绕远程数据展开。

### 8.1.2 基本术语

- HTTP 请求与响应。
- REST 风格接口。
- JSON 作为常见数据交换格式。

## 8.2 网络请求工具链

### 8.2.1 OkHttp

- 负责底层请求执行。
- 负责连接、超时、拦截器等基础能力。

### 8.2.2 Retrofit

- 负责把接口声明转换为更清晰的调用方式。
- 负责把网络层定义从业务逻辑中分离出来。

## 8.3 从请求到界面状态

### 8.3.1 成功、失败与加载态

- 不要只处理成功。
- 界面必须显式表达“正在加载”“加载失败”“暂无数据”等状态。

### 8.3.2 为什么网络请求不能直接塞进界面回调

- 界面层职责应该尽量保持清晰。
- 网络调用需要考虑取消、重试、异常和状态恢复。

## 8.4 本章实践任务

- 定义一个最小远程接口。
- 发起网络请求并解析 JSON。
- 把结果展示在列表界面中，并处理加载失败场景。

## 8.5 小结

本章引入了 Android 应用最基础的远程数据视角。学完本章后，读者应能建立“界面不是直接面向静态数据，而是面向异步远程状态”的认识。
