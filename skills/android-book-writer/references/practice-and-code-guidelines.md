# Practice and Code Guidelines

Use this reference when examples, exercises, or implementation guidance are too thin.

## When to add code

Add code when the topic is better understood by seeing a minimal implementation, such as lifecycle handling, ViewModel wiring, Room setup, navigation calls, or permission requests.

Do not add code just to satisfy a template. For conceptual topics, a clear scenario, decision table, or observation task may teach better.

## Minimal example pattern

Before the snippet, state:

- What the snippet demonstrates
- Which file or layer it belongs to
- Which prerequisite assumptions the reader should know

After the snippet, explain:

- Which lines carry the teaching point
- What the reader should run or observe
- Which simplifications were made on purpose

## Practice task pattern

Every practical task should include:

- Starting point: what project or file the reader should already have
- Steps: what to change, in order
- Expected result: what behavior or UI change confirms success
- Self-check: one or two quick checks the reader can perform
- Debugging hints: only the most likely failure points

## Good exercise traits

- Small enough to finish in one sitting
- Directly tied to the chapter's main teaching point
- Produces an observable result
- Reuses the chapter example rather than starting a new app from zero

## Common failures to remove

- "Try it yourself" with no instructions
- Full-project dumps with no explanation
- Exercises that test memory instead of implementation
- Code snippets that depend on unseen setup without saying so
