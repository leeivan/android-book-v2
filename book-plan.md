# 写作计划

## 目标

按照 `SUMMARY.md` 的章节顺序重建并持续打磨一套适合长期维护的 Android 教程仓库，使章节、附录、写作规范和提示词都能在同一仓库中协同工作。

## 阶段划分

1. 完成仓库骨架重构。
2. 完成全书首轮正文重写。
3. 完成第 1 到第 11 部分的“正式出版版”深度扩写。
4. 回到全书做一致性修订、术语统一、案例串联和目录对齐。
5. 进入最终出版级校对与补充交叉引用阶段。

## 内容要求

- 正文优先采用现代 Android 实践。
- 旧 API 仅作为迁移说明或历史背景。
- 每章都应说明本章目标、前置知识、核心概念、最小案例和常见误区。
- 章节间保持同一条项目主线，避免内容割裂。

## 优先级

- 第一优先级：全书术语统一与目录、标题一致性。
- 第二优先级：章节之间的交叉引用、案例串联和重复表述压缩。
- 第三优先级：最终出版级校对，包括前后衔接、示例密度和附录可查阅性优化。
