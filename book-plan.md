# 写作计划

## 目标

按照 `SUMMARY.md` 的章节顺序重建一套适合长期维护的 Android 教程仓库，使章节、附录、写作规范和提示词都能在同一仓库中协同工作。

## 阶段划分

1. 完成仓库骨架重构。
2. 完成前言、基础篇和 UI 开发部分的首轮正文。
3. 完成数据、网络、架构与并发部分的首轮正文。
4. 完成系统组件、工程实践、发布与综合项目。
5. 回到全书做一致性修订、术语统一和案例串联。

## 内容要求

- 正文优先采用现代 Android 实践。
- 旧 API 仅作为迁移说明或历史背景。
- 每章都应说明本章目标、前置知识、核心概念、最小案例和常见误区。
- 章节间保持同一条项目主线，避免内容割裂。

## 优先级

- 第一优先级：基础篇、UI 开发、数据存储。
- 第二优先级：网络开发、应用架构、并发与异步。
- 第三优先级：系统组件、工程实践、发布与综合项目。
